15 — NEU ⭐ Morph Nicht TROTZ sondern WEGEN — Der Mechanismus

DER Erklärungs-Slide. Zeigt visuell, wie eine Halluzination entsteht — Schritt für Schritt. Wiederverwendet die Next-Token-Animation.

Animation — Halluzination entsteht live
Phase 1: Normaler Prompt wird getippt
  "Nenne mir die BGH-Entscheidung zu Softwarepatenten"

Phase 2: Das Modell beginnt zu antworten (Token für Token)
  "Gemäß" → "der" → "BGH" → "-" → "Entscheidung" → ...
  (Balken zeigen Wahrscheinlichkeiten, alles normal)

Phase 3: Jetzt kommt das Aktenzeichen...
  "BGH-Entscheidung ___"

  Balken zeigen:
    X     ████████████  38%    ← Das MUSTER "X ZR xx/xx"
    I     ████████      24%      ist statistisch häufig
    V     ███           12%      nach "BGH-Entscheidung"

  → "X" wird gewählt.

Phase 4: Nächstes Token...
  "X ___"
    ZR    ████████████  45%    ← Wieder: MUSTER, nicht Fakt
    ZB    ████████      28%
    ...

  → "ZR" wird gewählt.

Phase 5: Die Ziffern...
  "X ZR ___"
    47    ████          12%    ← Irgendeine plausible Zahl
    23    ████          11%      Keine davon ist "nachgeschlagen"
    18    ███           10%      Alle sind gleich wahrscheinlich
    ...

  → "47/19" wird generiert.

Phase 6: REVEAL
  ┌────────────────────────────────────────┐
  │  "X ZR 47/19"                         │
  │                                        │
  │  ⚠ Dieses Aktenzeichen wurde nicht    │
  │    nachgeschlagen. Es wurde GENERIERT  │
  │    — Wort für Wort, basierend auf      │
  │    Mustern.                            │
  │                                        │
  │  Das Modell hat kein Bücherregal.     │
  │  Es hat Muster.                        │
  └────────────────────────────────────────┘
⚡ Das ist der Kern: Das Modell halluziniert nicht, weil es kaputt ist. Es halluziniert, weil es genau das tut, wofür es trainiert wurde: das plausibelste nächste Wort vorhersagen. Plausibel ≠ wahr.

Transition: Die generierten Tokens morphen zusammen zum vollständigen Fake-Zitat, das dann zur nächsten Slide überleitet (Glitch-Reveal).
